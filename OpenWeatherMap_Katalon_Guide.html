Project Structure

After you copy all scripts into Katalon Studio, your project tree will look like this:

openweather_katalon/ │ ├── Profiles/ │ └── default.glbl ← Global Variables (API key, base URL, coordinates) │ ├── Test Cases/ │ └── API/ │ ├── TC_001_5Day_Forecast_JakartaSelatan ← Positive: 5-day forecast │ ├── TC_002_Current_Air_Pollution_JakartaSelatan ← Positive: air pollution │ ├── TC_003_Negative_InvalidAPIKey ← Negative: 401 scenario │ └── TC_004_Negative_InvalidCoordinates ← Negative: bad params │ ├── Test Suites/ │ └── TS_OpenWeatherMap_JakartaSelatan ← Runs all 4 TCs in order │ └── Reports/ └── TS_OpenWeatherMap_JakartaSelatan/ └── YYYYMMDD_HHmmss/ ├── *.html ← Visual HTML report ├── *.csv ← Raw data export └── JUnit_Report.xml ← For CI/CD integration
⚙️
Profiles/default.glbl
Stores all global variables: API key, base URL, lat/lon Jakarta Selatan, units. Edit here once — semua test case otomatis pakai.
🧪
Test Cases/API/
4 test cases Groovy script. Masing-masing berdiri sendiri — tidak perlu external keyword/library.
▶️
Test Suites/
1 test suite yang memanggil semua 4 TC secara berurutan. Run suite ini untuk eksekusi penuh.
📊
Reports/
Auto-generated setelah setiap run. HTML report + JUnit XML + CSV. Folder timestamped otomatis.

Setup & Configuration

Prerequisites

ToolVersionNotes
Katalon Studio8.x atau lebih baruDownload gratis di katalon.com/download
Java JDK8+ (bundled)Sudah include di dalam installer Katalon
Internet—Untuk hit api.openweathermap.org
OWM API KeyFree tierRegister di openweathermap.org → API Keys
💡
Cara dapat API Key gratis: Daftar di openweathermap.org → My Profile → API Keys → copy default key. Tunggu ~10 menit sampai aktif sebelum pertama kali dipakai.
1
Buka atau buat project di Katalon Studio
  • Klik File → New → Project
  • Pilih Type: API/Web Service
  • Beri nama: openweather_katalon
  • Klik OK
2
Set Global Variables

Di panel kiri Tests Explorer → Profiles → default → double-click → klik Add (+) untuk setiap variable:

NameValue
API_KEY'YOUR_API_KEY_HERE'
BASE_URL'https://api.openweathermap.org/data/2.5'
LAT_JAKARTA_SELATAN'-6.2615'
LON_JAKARTA_SELATAN'106.8106'
UNITS'metric'

Ganti YOUR_API_KEY_HERE dengan API key aslimu. Klik Save (Ctrl+S).

3
Buat folder dan paste Test Cases
  • Klik kanan Test Cases → New → Folder → nama: API
  • Klik kanan folder API → New → Test Case
  • Beri nama sesuai file (misal TC_001_5Day_Forecast_JakartaSelatan)
  • Klik tab Script di editor → Ctrl+A → Delete → Paste isi file .groovy
  • Klik Save (Ctrl+S)
  • Ulangi untuk TC_002, TC_003, TC_004
⚠️
Pastikan paste di tab Script, bukan tab Manual. Kalau di tab Manual, kode tidak akan terbaca sebagai Groovy.
4
Buat Test Suite
  • Klik kanan Test Suites → New → Test Suite
  • Nama: TS_OpenWeatherMap_JakartaSelatan → OK
  • Di editor Test Suite, klik tombol Add (+)
  • Cari dan tambahkan keempat TC satu per satu
  • Pastikan semua checked di kolom Run
  • Klik Save

How to Run Tests

Option A — Run Full Test Suite (Recommended)

1
Buka Test Suite
Di panel kiri Tests Explorer → expand Test Suites → double-click TS_OpenWeatherMap_JakartaSelatan
2
Klik Run ▶
Klik tombol Run ▶ di toolbar atas, atau klik kanan nama Test Suite → Run
3
Pilih execution profile
Di dialog yang muncul, pastikan Profile = default. Klik OK.
4
Monitor di Console & Log Viewer
Panel bawah akan menampilkan output real-time. Setiap assertion yang pass akan muncul tanda ✅.
✅
Jika semua passed, kamu akan melihat "✅ TC_00X PASSED" di akhir setiap test case di Log Viewer.

Option B — Run Test Case Satu per Satu

Berguna untuk debug satu test case spesifik:

  • Tests Explorer → Test Cases → API
  • Klik kanan test case yang diinginkan → Run

Option C — Run via Command Line (CI/CD)

# Windows
katalonc.exe -noSplash \
  -runMode=console \
  -projectPath="C:\path\to\openweather_katalon\openweather_katalon.prj" \
  -testSuitePath="Test Suites/TS_OpenWeatherMap_JakartaSelatan" \
  -executionProfile="default" \
  -browserType="API"

# macOS / Linux
./katalonc -noSplash \
  -runMode=console \
  -projectPath="/path/to/openweather_katalon/openweather_katalon.prj" \
  -testSuitePath="Test Suites/TS_OpenWeatherMap_JakartaSelatan" \
  -executionProfile="default" \
  -browserType="API"
ℹ️
katalonc adalah Katalon console executable, ada di folder instalasi Katalon Studio. Di Windows biasanya di C:\Katalon_Studio_Windows_64-8.x.x\

How to Get the Report

Katalon Studio auto-generate report setiap kali test suite selesai dijalankan. Ada 3 cara aksesnya:

Cara 1 — Langsung dari Katalon Studio (Paling Mudah)

1
Setelah run selesai, klik "Open Report"
Di panel bawah (Job Progress / Log Viewer), setelah semua TC selesai, Katalon otomatis tampilkan notifikasi "Test Suite execution finished" dengan link Open Report. Klik link tersebut.
2
Browser terbuka dengan HTML Report
Report visual terbuka otomatis di browser default kamu. Tampilan berisi summary passed/failed, durasi, dan detail per assertion.

Cara 2 — Via Tests Explorer

1
Buka folder Reports di Tests Explorer
Panel kiri → expand Reports → expand TS_OpenWeatherMap_JakartaSelatan
2
Pilih folder run terbaru
Folder bernama timestamp, contoh: 20250318_193400. Double-click folder untuk expand.
3
Buka file report
  • Double-click file .html → HTML report visual terbuka di browser
  • Double-click file .csv → buka di Excel untuk data mentah
  • File JUnit_Report.xml → untuk diimport ke Jenkins/GitHub Actions

Cara 3 — Akses Langsung di File Explorer (Windows)

C:\Users\{nama_user}\Katalon Studio\openweather_katalon\
  Reports\
    TS_OpenWeatherMap_JakartaSelatan\
      20250318_193400\          ← folder timestamp run
        TS_OpenWeatherMap_JakartaSelatan.html   ← buka ini di browser
        TS_OpenWeatherMap_JakartaSelatan.csv
        JUnit_Report.xml
